# SaaS

SaaS (Software as a Service) é um modelo de entrega de software onde aplicações são disponibilizadas pela internet como um serviço.

Nesse modelo, o usuário não precisa instalar, manter ou atualizar o software localmente.

O acesso normalmente ocorre através de:

* navegador;
* aplicativo móvel;
* cliente web.

## O que significa SaaS?

A sigla significa:

```txt
Software as a Service
```

Em português:

```txt
Software como Serviço
```

O cliente paga pelo uso do sistema e o fornecedor é responsável pela infraestrutura, manutenção e evolução da aplicação.

## Como funciona?

Em um software tradicional:

```txt
Cliente
↓
Compra software
↓
Instala localmente
↓
Gerencia atualizações
```

Em um SaaS:

```txt
Cliente
↓
Acessa pela internet
↓
Utiliza o sistema
```

Toda a infraestrutura fica sob responsabilidade do fornecedor.

## Exemplos de SaaS

Aplicações bastante conhecidas:

* Google Drive;
* Gmail;
* Trello;
* Slack;
* Notion;
* Salesforce;
* Jira.

O usuário apenas utiliza o serviço.

## Características

### Acesso pela internet

O sistema é acessado online.

### Assinatura

Normalmente utiliza cobrança recorrente.

Exemplo:

```txt
Mensal

Anual
```

### Atualizações centralizadas

O fornecedor atualiza o sistema sem necessidade de ação do cliente.

### Escalabilidade

Novos usuários podem ser adicionados facilmente.

## Exemplo prático

Imagine um sistema de gestão empresarial.

Modelo tradicional:

```txt
Empresa compra licença
↓
Instala servidor
↓
Gerencia infraestrutura
```

Modelo SaaS:

```txt
Empresa cria conta
↓
Acessa pelo navegador
↓
Utiliza o sistema
```

## Benefícios para clientes

* Menor custo inicial;
* Sem instalação complexa;
* Atualizações automáticas;
* Acesso remoto;
* Escalabilidade.

## Benefícios para empresas

* Receita recorrente;
* Facilidade de distribuição;
* Atualizações centralizadas;
* Maior proximidade com clientes.

## SaaS e Cloud Computing

O crescimento do SaaS está diretamente ligado à computação em nuvem.

Exemplo:

```txt
AWS

Google Cloud

Azure
```

Essas plataformas permitem hospedar aplicações SaaS para milhares ou milhões de usuários.

## Multi-Tenant

Uma característica comum em SaaS é o modelo Multi-Tenant.

Nesse modelo:

```txt
Uma aplicação
↓
Vários clientes
```

Os clientes compartilham a mesma infraestrutura, mas possuem dados isolados.

## SaaS x Software Tradicional

### Software Tradicional

```txt
Compra única

Instalação local

Atualizações manuais
```

### SaaS

```txt
Assinatura

Acesso online

Atualizações automáticas
```

## SaaS x PaaS x IaaS

Cloud Computing costuma ser dividida em três modelos.

### SaaS

```txt
Usuário utiliza o software
```

Exemplo:

```txt
Google Drive

Slack

Notion
```

### PaaS

```txt
Usuário desenvolve aplicações
```

Exemplo:

```txt
Heroku

Google App Engine
```

### IaaS

```txt
Usuário gerencia infraestrutura
```

Exemplo:

```txt
AWS EC2

Google Compute Engine
```

## Arquitetura de um SaaS

Uma aplicação SaaS normalmente utiliza:

```txt
Frontend
↓
API
↓
Banco de Dados
↓
Cloud
```

Conforme cresce, pode incorporar:

* microsserviços;
* mensageria;
* filas;
* cache;
* observabilidade.

## Desafios

* Escalabilidade;
* Segurança;
* Disponibilidade;
* Isolamento de clientes;
* Custos de infraestrutura.

## Resumo

SaaS (Software as a Service) é um modelo de entrega de software onde aplicações são disponibilizadas pela internet como um serviço.

Os usuários acessam o sistema online, enquanto a empresa responsável gerencia infraestrutura, atualizações e manutenção, normalmente utilizando modelos de assinatura recorrente.
