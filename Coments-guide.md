# Comentários

**Comentários** são essenciais para tornar o código mais legível, explicar lógicas complexas, documentar funções, e ajudar na colaboração entre devs. Eles variam um pouco de sintaxe conforme a linguagem de programação.

## Tipos

1. Comentário de linha única: Usado para comentários curtos ou anotações rápidas.
2. Comentário de bloco / multiplas linhas: Usado para comentários maiores ou para desabilitar blocos de código temporariamente.
3. Comentários de documentação: Usado para documentação automática. Geralmente aparece acima de funções, classes ou metódos.

## Boas Práticas

  - Comente o "por que" algo está sendo feito e não só "o que"
  - Evite comentários óbvios
  - Mantenha os documentários atualizados com o código
  - Use comentários de documentações para APIs públicas e bibliotecas

## Por Linguagem

1. C / C++ / JAVA / Javascript
   - Para linha única: //
   - Para bloco: /*...*/

2. Go
   - //

3. html
   - <.!-- --> (retirar o .)

4. Python
   - Para linha única: #
   - Para bloco: "'...'" ou "..."
