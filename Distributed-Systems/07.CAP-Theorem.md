# CAP Theorem

CAP Theorem afirma que um sistema distribuído não pode garantir simultaneamente:

- Consistency;
- Availability;
- Partition Tolerance.

Em caso de falha de rede, é necessário escolher entre consistência ou disponibilidade.

## Consistency

Todos os nós veem os mesmos dados.

## Availability

O sistema sempre responde.

## Partition Tolerance

O sistema continua funcionando mesmo com falhas de comunicação.

## Exemplo

Rede dividida:

Servidor A
✕
Servidor B

O sistema precisa decidir:

Responder com possíveis inconsistências

ou

Bloquear operações.

## Resumo

CAP Theorem é um princípio fundamental que descreve os limites dos sistemas distribuídos.
