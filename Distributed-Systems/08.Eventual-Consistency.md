# Eventual Consistency

Eventual Consistency é um modelo onde os dados podem ficar temporariamente inconsistentes, mas eventualmente convergem para o mesmo estado.

## O problema

Em sistemas distribuídos:

Servidor A
↓
Atualiza dado

Servidor B
↓
Ainda não recebeu atualização

Durante um período existe inconsistência.

## Como funciona?

Atualização
↓
Replicação
↓
Sincronização
↓
Consistência

## Exemplo

Rede social.

Curtida realizada:

Usuário A vê imediatamente.

Usuário B pode visualizar alguns segundos depois.

## Benefícios

- Alta disponibilidade;
- Melhor escalabilidade;
- Menor latência.

## Desafios

- Dados temporariamente divergentes;
- Maior complexidade de negócio.

## Relação com CAP

Eventual Consistency normalmente surge quando sistemas priorizam:

Availability
+
Partition Tolerance

em vez de consistência imediata.

## Resumo

Eventual Consistency é um modelo de consistência onde os dados convergem ao mesmo estado ao longo do tempo, sendo amplamente utilizado em sistemas distribuídos modernos.
