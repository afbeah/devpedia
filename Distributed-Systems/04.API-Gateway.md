# API Gateway

API Gateway é um ponto único de entrada para múltiplos serviços.

Ele recebe requisições dos clientes e as direciona para os serviços corretos.

## O problema

Sem Gateway:

Cliente
├── User Service
├── Order Service
├── Payment Service

O cliente precisa conhecer todos os serviços.

## Solução

Cliente
↓
API Gateway
├── User Service
├── Order Service
└── Payment Service

O cliente conversa apenas com o Gateway.

## Responsabilidades

- Roteamento;
- Autenticação;
- Rate Limiting;
- Logs;
- Monitoramento.

## Benefícios

- Centralização;
- Segurança;
- Simplificação para clientes.

## Ferramentas

- Kong;
- NGINX;
- AWS API Gateway;
- Apigee.

## Resumo

API Gateway é uma camada intermediária que centraliza o acesso aos serviços de uma arquitetura distribuída.
