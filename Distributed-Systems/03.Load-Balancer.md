# Load Balancer

Load Balancer é um componente responsável por distribuir requisições entre múltiplas instâncias de uma aplicação.

Seu objetivo é evitar sobrecarga e aumentar a disponibilidade do sistema.

## O problema

Imagine:

Usuários
↓
Servidor Único

Se o servidor falhar:

Sistema indisponível

## Solução

Usuários
↓
Load Balancer
├── Servidor A
├── Servidor B
└── Servidor C

As requisições são distribuídas automaticamente.

## Benefícios

- Escalabilidade;
- Alta disponibilidade;
- Tolerância a falhas;
- Melhor utilização dos recursos.

## Estratégias

### Round Robin

Distribui alternadamente.

### Least Connections

Envia para o servidor com menos conexões.

### Weighted

Distribui conforme pesos definidos.

## Ferramentas

- NGINX;
- HAProxy;
- AWS ELB;
- Google Cloud Load Balancer.

## Resumo

Load Balancer é um mecanismo que distribui tráfego entre múltiplas instâncias para aumentar disponibilidade e escalabilidade.
