# Redis

Redis é um banco de dados em memória utilizado para armazenamento extremamente rápido de informações.

Seu nome significa:

Redis = Remote Dictionary Server

Embora seja classificado como banco de dados NoSQL, seu uso mais comum é como mecanismo de cache.

## O problema

Imagine uma aplicação realizando milhares de consultas por segundo.

Fluxo:

Usuário
↓
API
↓
Banco de Dados
↓
Resposta

Mesmo com índices e otimizações, o banco pode se tornar um gargalo.

## Como funciona?

Redis armazena informações diretamente na memória RAM.

Fluxo:

Usuário
↓
API
↓
Redis
↓
Resposta

A leitura ocorre muito mais rapidamente do que em bancos tradicionais.

## Exemplo prático

Consulta:

Buscar produto mais vendido

Primeira execução:

API
↓
Banco
↓
Redis

Próximas execuções:

API
↓
Redis

Sem necessidade de consultar o banco novamente.

## Principais usos

- Cache;
- Sessões;
- Rate Limiting;
- Filas;
- Contadores;
- Rankings.

## Estruturas de dados

Redis suporta:

- String;
- List;
- Set;
- Hash;
- Sorted Set.

## Benefícios

- Altíssima velocidade;
- Baixa latência;
- Fácil integração;
- Escalabilidade.

## Desafios

- Consumo de memória;
- Persistência limitada;
- Custo maior para grandes volumes.

## Redis e Cache

Cache é um conceito.

Redis é uma ferramenta utilizada para implementar esse conceito.

## Resumo

Redis é um banco de dados em memória amplamente utilizado para cache, filas, sessões e processamento de alta performance.
