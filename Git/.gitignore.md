# .gitignore

É um arquivo de texto colocado na raíz do seu repositório git. Ele serve para informar ao git quais arquivos ou diretórios devem ser ignorados e, consequentemente, não serão adicionados ao controle de versão.

## Por que usar?

1. Organização: Mantém o repositório limpo, evitando a inclusão de arquivos temporários, configurações locais, arquivos gerados automaticamente (como logs, builds, etc) que não são relevantes para o projeto em si.
2. Economia de espaços: Evita que arquivos grandes e desnecessários sejam adicionados ao repositório, economizando espaço e otimizando o processo de clonagem.
3. Ecita conflitos: Reduz a chance de conflitos ao mesclar diferentes versões do projeto, já que arquivos ignorados não serão considerados
4. Melhora a colaboração: Garante que todos os desenvolvedores do projeto tenham a mesma visão sobre quais arquivos devem ser versionados.

## Como funciona?

O git verifica o conteúdo do arquivo antes de adicionar um novo arquivo ao stage, se o nome do arquivo corresponde a um dos padrões definidos no .gitignore, ele será ignorado.
