# Commit Semântico

É uma prática de escrita de mensagens de commit de forma padronizada e significativa, ajudando a manter um histórico de commits mais legivéis e informativos. Essa prática é especialmente útil em projetos de software colaborativos, pois facilita a compreensão do histórico de mudanças por todos os membros da equipe. 

## Tipos

1. Feat: Uma nova funcionalidade para o usuário.
   - "feat: "

2. Fix: Correção de um bug.
   - "fix: "
  
3. Chore: Atualização de tarefas de rotina ou manutenção que não afetam a aplição, fonte ou testes. (mudanças nos arquivos de configuração do build ou no gerenciados de pacotes)
   - "chore: "

4. Docs: Mudanças na documentação
   - "docs: "

5. Style: Mudanças que não afetam o significado do código (formatação, espaços em branco)
   - "style: "

6. Refactor: Mudanças de código que não corrige um bug nem adiciona uma funcinalidade (refatoração de código)
   - "refactor: "

7. Perf: Mudança de código que melhora a performance.
   - "perf: "
     
8. Test: Adição ou correção de testes.
   - "test: "

9. Build: Mudanças que afetam o sistema de build ou de dependências externas. (npm, guild)
    - "build: "
  
10. Ci: Mudanças no arquivo e scripts de configuração do CI. (travis, circle)
    - "ci: "
   
11. Revert: Reversão de um commit anterior
    - " revert: "
