# Draft

O termo significa "rascunho" e pode ter diferentes significados dependendo do contexto.

1. Draft em documentos e projetos
   No contexto geral, um draft é uma versão inicial de um doc, projeto ou ideia que ainda está sendo desenvolvido. Ele pode passar por várias revisões antes de ser finalizado.
   > ex: Um artigo ou relatório em fase de revisão, um contrato ainda não assinado.

2. Draft no github (pull request e issues)
   No github, um draf pull request é um PR que ainda não esta pronto para ser revisado ou mesclado. Ele permite que você compartilhe seu progresso sem notificar os revisores prematuramente.
   > ex: criar um Draft PR para que outros desenvolvedores possam acompanhar o progresso antes da finalização.

3. Drafts em APIS (especificações)
  Na especificação *Open Api (swagger)*, um draft pode se referir a uma versão inicial de uma documentação de API.
