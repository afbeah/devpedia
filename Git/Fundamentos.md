# Fundamentos

## O que é controle de versão?

É uma técnica que ajuda o código-fonte de uma aplicação, o seu papel é registrar todas as modificações de código, podendo também reverter as mesmas. 

Criar versão de um software em diferentes estágios , podendo alterar facilmente entre elas.

Cada membro da equipe pode trabalhar em uma versão diferente. Há ferramentas para trabalhar o controle de versão como Git e Sun.

## O que é Git?

É o sistema de controle de versão mais utilizado no mundo atualmente. O Git é baseado em repositórios, que que contém todas as versões do código e também as cópias de cada dev.

Todas as operações do git são otimizadas para ter alto desempenho e todos os objetos do git são protegidos como criptografia para evitar alterações indévidas e maliciosas.

O Git é um projeto Open-Source.

## O que é um repositório?

É onde o código vai ser armazenado, na maioria das vezes cada projeto tem um repositório (repo).

Quando criamos um repo estamos iniciando um projeto.

O repo pode ir para servidores que são especializados em gerenciar repos, como Github, Gitlab, Bitbucket. Cada dev pode baixar o repo e criar versões diferentes em sua máquina (local).

## O que é Github?

É um serviço para gerenciar repositórios, gratuito e amplamente utilizado. 

Podemos enviar nossos projetos para o Github e diponibilizá-los para outros devs. O Github é gratuito tanto para projetos públicos como para projetos privados.

## Markdown

É uma forma de adicionar estilo a textos na web, o arquivo **Readme.md** aceita markdown.

Será possível exibir trechos de código, links, imagens e muito mais. Dando uma melhor experiência para o usuário nas suas docs.
