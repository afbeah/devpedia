# Scripts

Script programming são arquivos de código utilizados para automatizar tarefas, executar comandos ou processar informações.

Normalmente, scripts são executados diretamente por um interpretador, sem necessidade de compilação prévia tradicional.

## Para que servem?

Scripts são muito usados para:

- automação;
- manipulação de arquivos;
- execução de comandos;
- integração entre sistemas;
- build e deploy;
- tarefas repetitivas.

## Características

- Execução rápida;
- Fácil automação;
- Desenvolvimento ágil;
- Integração com sistemas;
- Flexibilidade.

## Linguagens comuns para scripts

- JavaScript;
- Python;
- Bash;
- PowerShell;
- Shell Script.

### Exemplo simples em JavaScript

```JavaScript
console.log("Olá Mundo!")
```

## Script vs Programa

|              Script              |           Programa           |
|              :---:               |             :---:            |
|     Geralmente Interpretado      |      Pode ser compilado      | 
|        Simples e Rápido          |     Robusto e eficiente      |
|         Execução imediata        |        Estrutura maior       |
|    Automação e tarefas simples   |      Aplicação complexa      |

## Exemplos de uso

- Scripts de deploy;
- Scripts de build;
- Automação de backups;
- Scripts npm;
- Processamento de arquivos;
- Rotinas de CI/CD.

## Benefícios

- Redução de tarefas manuais;
- Agilidade;
- Automação;
