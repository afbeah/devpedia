# ServiceNow

ServiceNow é uma plataforma utilizada para gerenciamento e automação de processos de TI e negócios.

Ela é amplamente utilizada por empresas para centralizar solicitações, incidentes, mudanças, catálogos de serviços e fluxos de trabalho.

## O que é o ServiceNow?

Imagine uma empresa com milhares de funcionários.

Todos os dias surgem solicitações como:

* acesso a sistemas;
* criação de usuários;
* problemas em aplicações;
* solicitação de equipamentos;
* alterações em ambientes;
* aprovação de mudanças.

Controlar tudo isso por e-mail seria complexo.

O ServiceNow centraliza esses processos em uma única plataforma.

## Principais conceitos

### Incidente (Incident)

Um incidente representa uma falha ou interrupção em um serviço.

Exemplos:

* sistema fora do ar;
* erro em uma aplicação;
* usuário sem acesso.

Objetivo:

```txt
Restaurar o serviço o mais rápido possível.
```

### Requisição (Request)

Uma requisição representa uma solicitação de serviço.

Exemplos:

* solicitar notebook;
* pedir acesso a uma aplicação;
* criar uma conta.

Objetivo:

```txt
Atender uma necessidade do usuário.
```

### Change (Mudança)

Representa uma alteração planejada em um ambiente.

Exemplos:

* deploy em produção;
* atualização de infraestrutura;
* alteração de configuração.

Objetivo:

```txt
Executar mudanças de forma controlada e segura.
```

### Catálogo de Serviços

É uma lista de serviços disponíveis para os usuários.

Exemplos:

* solicitar acesso;
* abrir chamado;
* requisitar equipamento;
* solicitar suporte.

## Fluxo simplificado

```txt
Usuário
↓
Abre solicitação
↓
ServiceNow
↓
Time responsável
↓
Atendimento
↓
Conclusão
```

## Automação

Uma das grandes vantagens do ServiceNow é a automação de processos.

Exemplo:

```txt
Funcionário solicita acesso
↓
Gestor aprova
↓
Sistema cria a solicitação
↓
Equipe executa a ação
↓
Usuário é notificado
```

Sem necessidade de múltiplos e-mails ou controles manuais.

## Benefícios

* Centralização de solicitações;
* Rastreabilidade;
* Automação de processos;
* Controle de mudanças;
* Gestão de incidentes;
* Histórico de atendimentos;
* Maior organização operacional.

## Onde é utilizado?

O ServiceNow é muito comum em:

* grandes empresas;
* bancos;
* empresas de tecnologia;
* telecomunicações;
* operações de TI;
* centros de suporte.

## Relação com DevOps

Embora não seja uma ferramenta de desenvolvimento, o ServiceNow frequentemente participa de processos relacionados a:

* deploys;
* mudanças em produção;
* gestão de incidentes;
* aprovações;
* operações de TI.

Por isso é comum desenvolvedores e engenheiros de software terem contato com a plataforma.

## Resumo

ServiceNow é uma plataforma utilizada para gerenciamento e automação de processos corporativos, especialmente na área de TI.

Ela ajuda empresas a controlar incidentes, solicitações, mudanças e fluxos de trabalho, proporcionando maior organização, rastreabilidade e eficiência operacional.
