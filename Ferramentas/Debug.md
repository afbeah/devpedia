# Debug

*Debugger* é o processo de identificar, analisar e corrigir erros (bugs) em um programa.

Ferramentas de debug permitem executar a aplicação passo a passo, analisando o comportamento do código em tempo real.

## Como funciona?

O debug normalmente funciona através de:

- breakpoints;
- inspeção de variáveis;
- execução controlada;
- análise do fluxo do programa.

## Breakpoints

Breakpoints são pontos de interrupção onde a execução do código é pausada temporariamente.

Isso permite:

- visualizar variáveis;
- acompanhar chamadas de função;
- entender o fluxo da aplicação;
- identificar erros.

## Fluxo básico de debug

- Definir breakpoints;
- Executar o programa em modo debug;
- Inspecionar variáveis;
- Navegar linha por linha;
- Encontrar o problema.

## Comandos comuns

|        Comando       |           Função           |
|         :---:        |           :---:            |
|    Step Over (F8)    |    Executa a linha atual   |
|    Step Into (F7)    |   Entra na função chamada  | 
|  Step Out (Shift+F8) |     Sai da função atual    |
|      Resume (F9)     |      Continua execução     |

## Ferramentas comuns

- Visual Studio Code
- IntelliJ IDEA
- Eclipse;
- Chrome DevTools;
- Delve (Go);
- GDB.

## Benefícios

- Identificação rápida de bugs;
- Melhor entendimento do código;
- Facilita troubleshooting;
- Reduz tentativa e erro;
- Ajuda na manutenção.
