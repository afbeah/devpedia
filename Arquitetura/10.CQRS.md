# CQRS

CQRS (Command Query Responsibility Segregation) é um padrão arquitetural que separa operações de leitura e escrita.

A sigla significa:

```txt
Command Query Responsibility Segregation
```

Em português:

```txt
Segregação de Responsabilidades entre Comandos e Consultas
```

## O problema

Em sistemas tradicionais, a mesma estrutura é utilizada para:

* consultar dados;
* alterar dados.

Exemplo:

```txt
User
↓
Criar
Consultar
Atualizar
Excluir
```

Com o crescimento da aplicação, isso pode gerar complexidade.

## A solução

O CQRS separa:

### Commands

Responsáveis por alterar dados.

Exemplos:

```txt
Criar Pedido

Atualizar Usuário

Cancelar Pagamento
```

### Queries

Responsáveis apenas por consultar dados.

Exemplos:

```txt
Listar Usuários

Consultar Pedidos

Gerar Relatórios
```

## Estrutura

```txt
Commands
↓
Write Model

Queries
↓
Read Model
```

## Benefícios

* Melhor escalabilidade;
* Maior organização;
* Otimização de leitura e escrita;
* Integração com Event-Driven.

## Desafios

* Complexidade adicional;
* Duplicação de modelos;
* Consistência eventual.

## CQRS e Microsserviços

CQRS aparece frequentemente em sistemas distribuídos e arquiteturas orientadas a eventos.

Fluxo:

```txt
Command
↓
Evento
↓
Read Model atualizado
```

## Resumo

CQRS é um padrão arquitetural que separa operações de leitura e escrita para melhorar organização, escalabilidade e desempenho de sistemas complexos.
