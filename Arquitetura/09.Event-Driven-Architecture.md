# Event-Driven Architecture

Event-Driven Architecture (EDA) é um estilo arquitetural onde componentes se comunicam através de eventos.

Ao invés de um sistema chamar diretamente outro sistema, ele publica eventos que podem ser consumidos por diferentes aplicações.

## O que é um Evento?

Um evento representa algo que aconteceu no sistema.

Exemplos:

```txt
Pedido criado

Pagamento aprovado

Usuário cadastrado

Produto atualizado
```

O evento descreve um fato ocorrido.

## Como funciona?

Imagine um e-commerce.

Quando um pedido é criado:

```txt
Pedido criado
↓
Evento publicado
```

Outros sistemas podem reagir:

```txt
Pedido criado
├── Notificação
├── Faturamento
├── Estoque
└── Analytics
```

Nenhum deles precisa conhecer os demais.

## Componentes principais

### Producer

Responsável por gerar eventos.

Exemplo:

```txt
Order Service
↓
Pedido criado
```

### Broker

Responsável por distribuir eventos.

Exemplos:

* RabbitMQ;
* Kafka;
* Google Pub/Sub.

### Consumer

Responsável por processar eventos.

Exemplo:

```txt
Notification Service
```

## Fluxo

```txt
Producer
↓
Evento
↓
Broker
↓
Consumers
```

## Benefícios

* Baixo acoplamento;
* Escalabilidade;
* Flexibilidade;
* Integração facilitada.

## Desafios

* Observabilidade;
* Rastreamento de eventos;
* Consistência eventual;
* Debug mais complexo.

## EDA e Microsserviços

Event-Driven é amplamente utilizado em arquiteturas de microsserviços.

Exemplo:

```txt
Order Service
↓
Evento
↓
Payment Service
↓
Evento
↓
Notification Service
```

## Resumo

Event-Driven Architecture é um modelo onde sistemas se comunicam através de eventos, permitindo maior desacoplamento, escalabilidade e integração entre componentes.
