# Microsserviços

Microsserviços (Microservices) são um estilo arquitetural onde uma aplicação é dividida em múltiplos serviços independentes.

Cada serviço é responsável por uma funcionalidade específica do negócio e pode ser desenvolvido, implantado e escalado separadamente.

## O que são Microsserviços?

Imagine um e-commerce.

Em vez de uma única aplicação contendo tudo:

```txt
Usuários
Pedidos
Pagamentos
Produtos
Notificações
```

cada funcionalidade torna-se um serviço independente.

```txt
User Service

Order Service

Payment Service

Notification Service
```

Cada serviço possui sua própria responsabilidade.

## Como funciona?

Os serviços se comunicam através de APIs, mensageria ou eventos.

Exemplo:

```txt
Order Service
↓
Solicita pagamento
↓
Payment Service
```

ou

```txt
Pedido criado
↓
Evento publicado
↓
Notification Service
```

Os serviços continuam independentes, mas trabalham juntos para formar o sistema.

## Objetivo dos Microsserviços

O principal objetivo é permitir que partes do sistema evoluam independentemente.

Exemplo:

```txt
Payment Service
↓
Nova funcionalidade
↓
Deploy apenas desse serviço
```

Sem necessidade de publicar toda a aplicação.

## Características

### Independência

Cada serviço pode evoluir separadamente.

### Deploy independente

Uma alteração não exige deploy do sistema inteiro.

### Escalabilidade individual

Exemplo:

```txt
Pagamentos
↓
Maior demanda
↓
Escalar apenas Payment Service
```

### Autonomia de equipes

Equipes podem trabalhar em serviços diferentes sem grandes conflitos.

## Exemplo de arquitetura

```txt
Frontend
│
├── User Service
├── Order Service
├── Product Service
├── Payment Service
└── Notification Service
```

Cada serviço possui sua própria lógica.

## Comunicação entre serviços

### API REST

```txt
Service A
↓
HTTP
↓
Service B
```

### Mensageria

```txt
Service A
↓
RabbitMQ
↓
Service B
```

### Eventos

```txt
Pedido Criado
↓
Evento
↓
Consumidores
```

## Banco de Dados

Uma prática comum é que cada serviço possua seu próprio banco.

Exemplo:

```txt
User Service
↓
Banco de Usuários

Order Service
↓
Banco de Pedidos
```

Isso reduz acoplamento entre serviços.

## Benefícios

### Escalabilidade

Permite aumentar recursos apenas onde necessário.

### Deploy independente

Atualizações mais rápidas e seguras.

### Resiliência

Uma falha pode impactar apenas parte do sistema.

### Flexibilidade tecnológica

Diferentes serviços podem utilizar tecnologias diferentes.

Exemplo:

```txt
User Service → Go

Payment Service → Java

Analytics Service → Python
```

## Desafios

### Complexidade

Microsserviços aumentam significativamente a complexidade da arquitetura.

### Comunicação distribuída

Problemas de rede passam a existir.

### Observabilidade

Monitoramento torna-se mais difícil.

### Consistência de dados

Manter dados sincronizados entre serviços pode ser complexo.

## Microsserviços e Processamento Assíncrono

Conceitos como:

* Mensageria;
* Filas;
* Worker;
* Retry;
* Logs;

são muito utilizados em arquiteturas de microsserviços.

Exemplo:

```txt
Order Service
↓
Publica evento
↓
Fila
↓
Worker
↓
Notification Service
```

Por isso esses conceitos costumam aparecer juntos.

## Microsserviços x Monolito

### Monolito

```txt
Uma aplicação

Um deploy

Um ambiente
```

### Microsserviços

```txt
Vários serviços

Deploy independente

Comunicação distribuída
```

## Quando utilizar?

Microsserviços costumam ser indicados para:

* sistemas grandes;
* múltiplas equipes;
* alto volume de acessos;
* necessidades de escalabilidade específicas.

## Quando evitar?

Projetos pequenos ou equipes reduzidas podem não se beneficiar da complexidade adicional.

Exemplo:

```txt
MVP

Startup inicial

Sistema simples
```

Nesses cenários, um monolito costuma ser uma solução melhor.

## Monólito Primeiro

Uma abordagem bastante comum é:

```txt
Monolito
↓
Crescimento
↓
Microsserviços
```

Muitas empresas iniciam com um monólito e migram para microsserviços apenas quando surgem necessidades reais.

## Exemplos conhecidos

Empresas como:

* Netflix;
* Amazon;
* Uber;
* Spotify;

utilizam arquiteturas baseadas em microsserviços.

## Resumo

Microsserviços são um estilo arquitetural onde uma aplicação é dividida em serviços independentes.

Essa abordagem permite maior escalabilidade, autonomia e flexibilidade, mas também aumenta a complexidade da arquitetura e da operação do sistema.
