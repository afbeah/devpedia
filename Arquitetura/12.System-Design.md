# System Design

System Design é o processo de planejar e estruturar sistemas de software para atender requisitos de negócio, desempenho, escalabilidade e confiabilidade.

Seu objetivo é definir como um sistema será construído antes da implementação.

## O que um System Design define?

Normalmente define:

* arquitetura;
* banco de dados;
* comunicação entre serviços;
* escalabilidade;
* monitoramento;
* segurança.

## Exemplo simples

Imagine um sistema de encurtamento de URLs.

Fluxo:

```txt
Usuário
↓
API
↓
Banco de Dados
↓
URL encurtada
```

À medida que o sistema cresce, novas preocupações surgem:

* cache;
* balanceamento de carga;
* filas;
* processamento assíncrono.

## Componentes comuns

### API

Recebe requisições dos usuários.

### Banco de Dados

Armazena informações.

### Cache

Reduz consultas repetidas.

Exemplo:

```txt
Redis
```

### Mensageria

Permite processamento assíncrono.

Exemplo:

```txt
RabbitMQ

Kafka
```

### Workers

Executam tarefas em segundo plano.

## Escalabilidade

Um dos principais objetivos do System Design.

Exemplo:

```txt
1.000 usuários
↓
10 milhões de usuários
```

O sistema deve continuar funcionando.

## Relação com outros conceitos

System Design reúne diversos assuntos:

```txt
Arquitetura
↓
Microsserviços
↓
Mensageria
↓
Filas
↓
Workers
↓
Banco de Dados
↓
Cache
```

## Benefícios

* Melhor planejamento;
* Maior escalabilidade;
* Redução de gargalos;
* Facilidade de evolução.

## Resumo

System Design é a disciplina responsável por planejar como sistemas serão estruturados para atender requisitos técnicos e de negócio, considerando desempenho, escalabilidade, disponibilidade e manutenção.
