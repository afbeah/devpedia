# Design Patterns

Design Patterns (Padrões de Projeto) são soluções reutilizáveis para problemas recorrentes encontrados no desenvolvimento de software.

Eles não são código pronto, mas modelos que ajudam a estruturar soluções de forma consistente.

## Por que utilizar?

Durante o desenvolvimento, alguns problemas aparecem repetidamente.

Exemplos:

* criação de objetos;
* comunicação entre componentes;
* organização de responsabilidades.

Os Design Patterns surgiram para fornecer soluções conhecidas para esses problemas.

## Categorias

### Criacionais

Responsáveis pela criação de objetos.

Exemplos:

* Singleton;
* Factory Method;
* Builder.

### Estruturais

Responsáveis pela organização de componentes.

Exemplos:

* Adapter;
* Facade;
* Decorator.

### Comportamentais

Responsáveis pela comunicação entre objetos.

Exemplos:

* Strategy;
* Observer;
* Command.

## Exemplos comuns

### Strategy

Permite trocar comportamentos em tempo de execução.

Exemplo:

```txt
PIX

Cartão

Boleto
```

Cada forma de pagamento possui sua própria estratégia.

### Factory

Centraliza a criação de objetos.

Exemplo:

```txt
PaymentFactory
↓
PIX

Cartão

Boleto
```

### Observer

Permite que múltiplos componentes sejam notificados quando algo acontece.

Exemplo:

```txt
Pedido criado
↓
Notificar consumidores
```

## Benefícios

* Reutilização;
* Padronização;
* Facilidade de manutenção;
* Melhor comunicação entre desenvolvedores.

## Resumo

Design Patterns são soluções reutilizáveis para problemas recorrentes de desenvolvimento de software, ajudando a criar sistemas mais organizados e fáceis de manter.
