# AI Architecture

Uma arquitetura de Inteligência Artificial é a organização dos componentes responsáveis por permitir que um modelo de linguagem compreenda solicitações, recupere informações, execute ações e produza respostas para os usuários.

Seu principal objetivo é integrar modelos de IA, bases de conhecimento e sistemas externos de forma organizada, escalável e eficiente.

Em outras palavras:

> Uma aplicação de IA moderna é composta por diversos componentes trabalhando em conjunto, e não apenas por um LLM.

## O problema

Imagine um usuário fazendo a seguinte solicitação:

```txt
Analise meu currículo, encontre vagas compatíveis, adapte meu currículo para cada vaga e envie um resumo das alterações realizadas.
```

Para executar essa tarefa, a aplicação precisa:

- compreender o pedido;
- acessar documentos;
- pesquisar vagas;
- consultar APIs;
- adaptar o currículo;
- organizar todas as informações;
- gerar uma resposta.

Um único LLM não consegue realizar todas essas tarefas sozinho.

## Arquitetura Geral

Uma aplicação moderna de IA normalmente segue o seguinte fluxo:

```txt
                 Usuário
                     │
                     ▼
             Prompt Engineering
                     │
                     ▼
                 AI Agent
         ┌───────────┼───────────┐
         ▼                       ▼
       RAG                     MCP
         │                       │
         ▼                       ▼
Vector Database             Ferramentas
         ▲                       │
         │                       ▼
    Embeddings           Function Calling
         └───────────┬───────────┘
                     ▼
                    LLM
                     │
              Context Window
                     │
                 Inference
                     │
                     ▼
                 Resposta
```

Cada componente possui uma responsabilidade específica.

## Papel de Cada Componente

### LLM

É o núcleo da aplicação.

Responsável por compreender a solicitação e gerar a resposta.

---

### Prompt Engineering

Estrutura as instruções enviadas ao modelo.

Fornece contexto, restrições e objetivos.

---

### Context Window

Define a quantidade máxima de informações que o modelo consegue considerar durante uma interação.

---

### Inference

Representa o processo de geração da resposta pelo modelo.

---

### RAG

Recupera documentos relevantes para enriquecer o contexto enviado ao LLM.

---

### Embeddings

Transformam textos em vetores numéricos que representam seu significado.

---

### Vector Database

Armazena e pesquisa os embeddings utilizando busca por similaridade.

---

### Function Calling

Permite que o modelo execute funções externas.

Exemplos:

- consultar APIs;
- acessar bancos de dados;
- enviar e-mails;
- criar pedidos.

---

### MCP

Padroniza a comunicação entre modelos de IA e ferramentas externas.

Facilita integrações com diversos sistemas.

---

### AI Agent

Coordena todo o fluxo.

Planeja tarefas, escolhe ferramentas e organiza a execução.

---

### A2A

Permite que diferentes AI Agents trabalhem em conjunto.

Cada agente pode ser especializado em uma determinada responsabilidade.

## Fluxo de Execução

Imagine uma aplicação corporativa.

```txt
Usuário
↓
Solicitação
↓
AI Agent
↓
RAG
↓
Busca documentos
↓
MCP
↓
Consulta sistemas
↓
Function Calling
↓
Executa ações
↓
LLM
↓
Inference
↓
Resposta
```

Perceba que o LLM representa apenas uma parte da arquitetura.

Grande parte da inteligência da aplicação está na integração entre seus componentes.

## Benefícios

### Escalabilidade

Novos componentes podem ser adicionados sem modificar toda a aplicação.

### Modularidade

Cada componente possui uma responsabilidade específica.

### Reutilização

Os mesmos componentes podem ser utilizados em diferentes aplicações.

### Facilidade de manutenção

Alterações em um componente tendem a impactar menos os demais.

## Desafios

### Complexidade

Quanto maior a arquitetura, maior a necessidade de planejamento.

### Observabilidade

É importante monitorar todo o fluxo da aplicação.

### Segurança

O acesso a documentos e ferramentas deve ser cuidadosamente controlado.

### Custos

Cada chamada ao LLM, banco vetorial ou ferramenta externa possui custo computacional.

## Relação com o DevPedia

```txt
LLM

├── Inference
├── Fine-Tuning
├── RAG
│   ├── Embeddings
│   └── Vector Database
├── Function Calling
├── AI Agent
├── MCP
└── A2A
```

Todos esses componentes trabalham em conjunto para construir aplicações modernas baseadas em Inteligência Artificial.

## Resumo

Uma arquitetura moderna de IA combina modelos de linguagem, recuperação de conhecimento, bancos vetoriais, agentes inteligentes e integrações com sistemas externos para criar aplicações capazes de compreender solicitações, acessar informações, executar ações e gerar respostas de forma inteligente.

Mais do que um único modelo, aplicações de IA são compostas por diversos componentes especializados que trabalham em conjunto para entregar uma experiência completa ao usuário.
