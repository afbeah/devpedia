# Vector Database

Um Vector Database (Banco de Dados Vetorial) é um banco de dados projetado para armazenar, indexar e pesquisar vetores de alta dimensão.

Seu principal objetivo é permitir buscas por similaridade entre informações, tornando possível localizar documentos com significados semelhantes, mesmo quando utilizam palavras diferentes.

Em outras palavras:

> Um Vector Database é responsável por armazenar os Embeddings utilizados por aplicações de Inteligência Artificial.

## O problema

Após gerar os Embeddings de milhares de documentos, surge uma nova necessidade:

```txt
Onde armazenar esses vetores?
```

Um banco de dados relacional tradicional consegue armazenar números, porém não foi projetado para realizar buscas por similaridade em milhões de vetores.

Imagine uma empresa com:

- 500.000 documentos;
- milhões de embeddings;
- milhares de consultas por minuto.

Pesquisar manualmente todos esses vetores seria inviável.

## Como funciona?

Primeiro os documentos são transformados em Embeddings.

Fluxo:

```txt
Documento
↓
Embedding Model
↓
Vetor
↓
Vector Database
```

Quando um usuário realiza uma pergunta:

```txt
Pergunta
↓
Embedding
↓
Vector Database
↓
Vetores semelhantes
↓
Documentos encontrados
```

O banco retorna os documentos semanticamente mais próximos da pergunta.

## Exemplo

Documento:

```txt
Como solicitar férias?
```

Embedding:

```txt
[0.18, -0.42, 0.91, ...]
```

Esse vetor é armazenado no banco vetorial.

Pergunta:

```txt
Como pedir dias de descanso?
```

O sistema gera um novo embedding.

Em seguida:

```txt
Embedding da pergunta
↓
Vector Database
↓
Busca por similaridade
↓
Documento encontrado
```

Mesmo utilizando palavras diferentes, o sistema encontra o documento correto porque os significados são semelhantes.

## Busca Vetorial

Ao contrário de bancos relacionais, que normalmente utilizam igualdade:

```sql
WHERE nome = 'João'
```

um Vector Database procura:

```txt
Qual vetor é mais parecido com este?
```

Essa técnica é conhecida como:

```txt
Busca por Similaridade
```

ou

```txt
Nearest Neighbor Search
```

## Indexação Vetorial

Para acelerar as buscas, os bancos vetoriais utilizam estruturas de indexação especializadas.

Entre os algoritmos mais comuns estão:

- HNSW (Hierarchical Navigable Small World);
- IVF (Inverted File Index);
- PQ (Product Quantization).

Esses algoritmos tornam possível pesquisar milhões de vetores em poucos milissegundos.

## Principais Bancos Vetoriais

Algumas das soluções mais utilizadas atualmente:

- Pinecone;
- Weaviate;
- Milvus;
- Qdrant;
- Chroma;
- pgvector (PostgreSQL);
- Elasticsearch (Vector Search).

Cada uma possui características específicas de desempenho, escalabilidade e integração.

## Exemplo em um Sistema RAG

Fluxo completo:

```txt
Documentos
↓
Embeddings
↓
Vector Database

──────────────

Usuário
↓
Pergunta
↓
Embedding
↓
Vector Database
↓
Documentos semelhantes
↓
LLM
↓
Resposta
```

Perceba que o LLM nunca pesquisa diretamente os documentos.

Quem realiza essa busca é o banco vetorial.

## Benefícios

### Busca semântica

Permite localizar documentos pelo significado.

### Alta performance

Projetado para consultas vetoriais em grande escala.

### Escalabilidade

Suporta milhões de documentos e vetores.

### Melhor qualidade das respostas

Fornece contexto relevante para o LLM.

## Limitações

### Consumo de armazenamento

Cada documento gera um ou mais vetores.

### Atualização contínua

Novos documentos precisam gerar novos embeddings.

### Dependência da qualidade dos embeddings

Embeddings ruins resultam em buscas ruins.

## Quando utilizar?

Vector Databases costumam ser utilizados em:

- sistemas RAG;
- chatbots corporativos;
- assistentes virtuais;
- motores de busca inteligentes;
- recomendação de conteúdo;
- busca por imagens;
- busca por áudio e vídeo.

## Resumo

Um Vector Database é um banco de dados especializado no armazenamento e pesquisa de vetores.

Ele permite realizar buscas por similaridade entre documentos utilizando seus Embeddings, sendo um dos principais componentes de arquiteturas RAG e aplicações modernas de Inteligência Artificial.
