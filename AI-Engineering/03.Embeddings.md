# Embeddings

Embeddings são representações numéricas de textos, palavras ou documentos que preservam seus significados semânticos.

Seu principal objetivo é permitir que computadores comparem informações com base em contexto e significado, e não apenas por palavras iguais.

Em outras palavras:

> Embeddings transformam texto em números que representam significado.

## O problema

Imagine uma base de conhecimento contendo o seguinte documento:

```txt
Como solicitar férias?
```

Agora um usuário pergunta:

```txt
Como pedir alguns dias de descanso?
```

Embora as palavras sejam diferentes, ambas possuem praticamente o mesmo significado.

Uma busca tradicional baseada apenas em palavras pode não encontrar o documento correto.

## Como funciona?

Antes que os documentos sejam armazenados, eles são convertidos em vetores numéricos.

Fluxo:

```txt
Documento
↓
Embedding Model
↓
Vetor Numérico
```

Quando o usuário realiza uma pergunta:

```txt
Pergunta
↓
Embedding Model
↓
Vetor Numérico
```

O sistema compara os vetores.

Quanto mais próximos eles estiverem, maior a similaridade entre seus significados.

## Exemplo

Documento:

```txt
Como solicitar férias?
```

Embedding:

```txt
[0.12, -0.83, 0.44, ...]
```

Pergunta:

```txt
Como pedir dias de descanso?
```

Embedding:

```txt
[0.10, -0.79, 0.41, ...]
```

Mesmo utilizando palavras diferentes, os vetores permanecem próximos porque representam ideias semelhantes.

## Busca Semântica

Os embeddings tornam possível a chamada busca semântica.

Enquanto uma busca tradicional procura palavras iguais:

```txt
"férias"
```

A busca semântica procura documentos com significado semelhante.

Exemplo:

```txt
Férias

Folga

Descanso

Licença
```

Mesmo utilizando palavras diferentes, o sistema consegue localizar documentos relacionados.

## Como os Embeddings são utilizados?

Fluxo:

```txt
Documentos
↓
Embeddings
↓
Vector Database
```

Quando uma pergunta é realizada:

```txt
Pergunta
↓
Embedding
↓
Busca por similaridade
↓
Documentos relevantes
```

Esses documentos são enviados ao LLM.

## Exemplo em um sistema RAG

Fluxo completo:

```txt
Usuário
↓
Pergunta
↓
Embedding
↓
Vector Database
↓
Documentos semelhantes
↓
LLM
↓
Resposta
```

Perceba que o LLM não realiza a busca.

Quem encontra os documentos relevantes é o mecanismo baseado em embeddings.

## Benefícios

### Busca inteligente

Não depende apenas de palavras idênticas.

### Melhor recuperação de contexto

Documentos semelhantes são encontrados mesmo utilizando sinônimos.

### Melhor desempenho em RAG

Aumenta significativamente a qualidade das respostas.

### Flexibilidade

Permite trabalhar com diferentes idiomas e formas de escrita.

## Limitações

### Dependência do modelo

Embeddings de baixa qualidade produzem buscas menos precisas.

### Custo computacional

Gerar embeddings para grandes volumes de documentos pode consumir recursos.

### Atualização

Novos documentos precisam ter seus embeddings gerados antes de serem pesquisados.

## Ferramentas Comuns

Modelos de Embeddings:

* OpenAI Embeddings;
* BGE (BAAI General Embedding);
* Sentence Transformers;
* Cohere Embeddings;
* Google Embeddings.

## Resumo

Embeddings são representações numéricas de textos que preservam seu significado semântico.

Eles permitem que sistemas encontrem documentos semelhantes por contexto e intenção, sendo um dos principais componentes de arquiteturas RAG e aplicações modernas de Inteligência Artificial.
