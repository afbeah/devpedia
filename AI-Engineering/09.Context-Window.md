# Context Window

Context Window (Janela de Contexto) é a quantidade máxima de informações que um Large Language Model (LLM) consegue considerar durante uma interação.

Seu principal objetivo é definir o limite de contexto que o modelo pode utilizar para compreender uma solicitação e gerar uma resposta.

Em outras palavras:

> Context Window representa a "memória de curto prazo" de um LLM.

## O problema

Imagine um usuário enviando um documento com 800 páginas para análise.

Ou então uma conversa contendo centenas de mensagens.

O modelo consegue considerar todas essas informações?

A resposta depende do tamanho da sua janela de contexto.

Cada LLM possui um limite máximo de contexto.

Quando esse limite é ultrapassado, parte das informações deixa de ser considerada durante a geração da resposta.

## Como funciona?

Durante cada interação o modelo recebe:

- Prompt do usuário;
- Histórico da conversa;
- Contexto recuperado pelo RAG;
- Instruções do sistema;
- Ferramentas utilizadas.

Tudo isso ocupa espaço dentro da Context Window.

Fluxo:

```txt
Prompt

+

Histórico

+

Contexto (RAG)

+

Instruções

↓

Context Window

↓

LLM

↓

Resposta
```

Se o limite for atingido, parte dessas informações poderá ser removida.

## O que ocupa espaço?

Diversos elementos consomem a janela de contexto.

Exemplo:

- mensagens anteriores;
- documentos enviados;
- respostas do modelo;
- prompts do sistema;
- contexto recuperado pelo RAG;
- resultados de ferramentas.

Quanto maior o contexto enviado ao modelo, menor será o espaço disponível para novas informações.

## Tokens

A Context Window é medida em tokens.

Um token representa pequenas unidades de texto.

Por exemplo:

```txt
Olá, mundo!
```

Pode ser dividido em vários tokens.

Quanto maior o texto, maior será a quantidade de tokens utilizada.

## Exemplo

Imagine um modelo com:

```txt
Context Window

=

128.000 tokens
```

Durante uma conversa:

```txt
Sistema
20.000

+

Histórico
40.000

+

RAG
30.000

+

Prompt
10.000

=

100.000 tokens
```

Ainda restam aproximadamente:

```txt
28.000 tokens
```

para gerar a resposta.

## O que acontece quando o limite é atingido?

Quando a Context Window é ultrapassada:

- mensagens antigas podem ser descartadas;
- documentos podem ser truncados;
- parte do contexto recuperado pelo RAG pode não ser utilizada.

Como consequência, a qualidade das respostas pode diminuir.

## Context Window e RAG

Um erro comum é acreditar que basta recuperar centenas de documentos.

Na prática:

```txt
RAG
↓

Documentos

↓

Context Window
```

Somente os documentos que couberem dentro da janela poderão ser utilizados pelo modelo.

Por isso aplicações RAG normalmente recuperam apenas os documentos mais relevantes.

## Context Window e Prompt Engineering

Prompt Engineering também influencia diretamente a utilização da janela.

Prompts muito grandes:

- aumentam o consumo de tokens;
- reduzem espaço para documentos;
- diminuem espaço para respostas.

Um bom prompt procura fornecer contexto suficiente utilizando a menor quantidade possível de tokens.

## Benefícios de uma Janela Maior

### Mais contexto

Permite analisar documentos extensos.

### Conversas maiores

Mantém histórico por mais tempo.

### Melhor desempenho em RAG

Permite enviar mais contexto ao modelo.

### Maior capacidade de raciocínio

Modelos conseguem considerar mais informações simultaneamente.

## Limitações

### Maior custo

Mais tokens significam maior processamento.

### Maior latência

Conversas extensas tendem a consumir mais tempo.

### Limites físicos

Mesmo modelos modernos possuem um limite máximo de contexto.

## Exemplos de uso

Context Windows maiores são úteis para:

- análise de contratos;
- revisão de código;
- documentação técnica;
- chatbots corporativos;
- aplicações RAG;
- AI Agents.

## Resumo

Context Window é a quantidade máxima de informações que um Large Language Model consegue considerar durante uma interação.

Ela funciona como uma memória de curto prazo, limitando o volume de prompts, histórico, documentos e contexto que podem ser utilizados na geração de uma resposta.
