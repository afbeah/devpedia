# MCP (Model Context Protocol)

MCP (Model Context Protocol) é um protocolo aberto que padroniza a comunicação entre modelos de Inteligência Artificial e ferramentas externas.

Seu principal objetivo é permitir que LLMs e AI Agents utilizem diferentes serviços, APIs e bancos de dados através de uma interface padronizada.

Em outras palavras:

> O MCP é uma linguagem comum entre modelos de IA e ferramentas externas.

## O que significa MCP?

A sigla significa:

```txt
Model Context Protocol
```

Em português:

```txt
Protocolo de Contexto para Modelos
```

O protocolo define como uma aplicação disponibiliza informações e ferramentas para um modelo de IA.

## O problema

Imagine uma aplicação que deseja conectar um LLM com diversos sistemas.

Sem MCP:

```txt
LLM
├── API GitHub
├── API Slack
├── API Gmail
├── API PostgreSQL
├── API Jira
└── API Notion
```

Cada integração precisa ser desenvolvida individualmente.

Cada API possui autenticação, formatos e comportamentos diferentes.

Isso aumenta significativamente a complexidade da aplicação.

## Como funciona?

O MCP cria uma camada padronizada entre o modelo e os serviços.

Fluxo:

```txt
LLM
↓
MCP Client
↓
MCP Server
↓
Ferramentas
```

O modelo não precisa conhecer os detalhes de cada API.

Ele apenas solicita a utilização de uma ferramenta disponível.

## Componentes do MCP

Uma arquitetura MCP normalmente possui três elementos.

### Host

É a aplicação onde o modelo está sendo executado.

Exemplos:

- ChatGPT Desktop;
- Claude Desktop;
- IDEs;
- Aplicações corporativas.

### MCP Client

Responsável por comunicar o modelo com os servidores MCP.

Ele envia solicitações e recebe respostas.

### MCP Server

Disponibiliza ferramentas para o modelo.

Essas ferramentas podem acessar:

- APIs;
- bancos de dados;
- arquivos;
- sistemas internos;
- serviços externos.

## Exemplo

Imagine um usuário perguntando:

```txt
Quais Pull Requests estão abertas no meu GitHub?
```

Fluxo:

```txt
Usuário
↓
LLM
↓
MCP Client
↓
GitHub MCP Server
↓
GitHub API
↓
Lista de Pull Requests
↓
LLM
↓
Resposta
```

O modelo não conversa diretamente com o GitHub.

Toda a comunicação ocorre através do protocolo MCP.

## MCP x Function Calling

Embora estejam relacionados, possuem objetivos diferentes.

| Function Calling | MCP |
|------------------|-----|
| Executa funções definidas pela aplicação | Padroniza o acesso a ferramentas externas |
| Geralmente limitado ao sistema atual | Pode integrar múltiplos serviços |
| Implementação específica | Protocolo aberto |
| Cada aplicação define suas funções | Ferramentas seguem um padrão comum |

Function Calling é um recurso.

MCP é um protocolo.

## Benefícios

### Padronização

Ferramentas seguem uma interface comum.

### Reutilização

O mesmo servidor MCP pode ser utilizado por diferentes aplicações.

### Escalabilidade

Novas integrações podem ser adicionadas sem alterar o modelo.

### Menor complexidade

Reduz o esforço necessário para conectar IA a diferentes sistemas.

## Limitações

### Dependência do protocolo

As ferramentas precisam implementar o padrão MCP.

### Segurança

O acesso às ferramentas deve ser cuidadosamente controlado.

### Maturidade

O ecossistema ainda está em evolução e novas implementações surgem constantemente.

## Resumo

MCP (Model Context Protocol) é um protocolo aberto que padroniza a comunicação entre modelos de Inteligência Artificial e ferramentas externas.
