# Prompt Engineering

Prompt Engineering é a prática de projetar, estruturar e otimizar instruções enviadas a um Large Language Model (LLM) para obter respostas mais precisas, consistentes e úteis.

Seu principal objetivo é orientar o modelo para produzir resultados alinhados ao contexto e à necessidade da aplicação.

Em outras palavras:

> Prompt Engineering é a engenharia por trás da comunicação entre humanos e modelos de Inteligência Artificial.

## O problema

Imagine que um usuário envie a seguinte pergunta:

```txt
Explique Docker.
```

O modelo pode responder de diversas maneiras:

- Uma definição simples;
- Um tutorial completo;
- Uma explicação técnica;
- Uma comparação com máquinas virtuais.

Todas podem estar corretas.

O problema é que o modelo não sabe exatamente qual resposta o usuário espera.

## Como funciona?

O Prompt Engineering fornece contexto suficiente para orientar o comportamento do modelo.

Fluxo:

```txt
Usuário
↓
Prompt estruturado
↓
LLM
↓
Resposta
```

Quanto melhor o prompt, maior a probabilidade de obter uma resposta adequada.

## Exemplo

Prompt simples:

```txt
Explique Docker.
```

Resposta:

```txt
Docker é uma plataforma para criação e execução de containers.
```

Prompt estruturado:

```txt
Explique Docker para um estudante de Engenharia de Software.

Utilize linguagem simples.

Apresente exemplos.

Limite a resposta em cinco parágrafos.
```

A resposta torna-se muito mais adequada ao objetivo do usuário.

## Componentes de um Bom Prompt

Um prompt costuma ser composto por diferentes elementos.

### Contexto

Fornece informações importantes ao modelo.

Exemplo:

```txt
Você é um professor de Engenharia de Software.
```

### Objetivo

Define claramente o que deve ser realizado.

```txt
Explique o funcionamento do Docker.
```

### Restrições

Limitam o comportamento esperado.

Exemplo:

```txt
Utilize até cinco parágrafos.
```

### Formato da resposta

Especifica como a resposta deve ser apresentada.

Exemplo:

```txt
Responda utilizando Markdown.
```

## Técnicas Comuns

### Zero-Shot Prompting

O modelo recebe apenas a instrução.

```txt
Explique o que é Kubernetes.
```

### One-Shot Prompting

É apresentado um exemplo antes da solicitação.

```txt
Pergunta:
O que é Docker?

Resposta:
...

Agora responda:

O que é Kubernetes?
```

### Few-Shot Prompting

São fornecidos vários exemplos para orientar o modelo.

Quanto mais exemplos relevantes, maior tende a ser a consistência das respostas.

## Boas Práticas

- Defina claramente o objetivo;
- Forneça contexto suficiente;
- Utilize instruções específicas;
- Especifique o formato esperado;
- Evite ambiguidades;
- Divida tarefas complexas em etapas menores.

## Prompt Engineering em Aplicações

Em aplicações modernas, o prompt normalmente é construído dinamicamente.

Fluxo:

```txt
Usuário
↓
Aplicação
↓
Prompt
+
Contexto
+
Documentos do RAG
↓
LLM
↓
Resposta
```

Perceba que o usuário raramente envia diretamente o prompt completo.

A aplicação é responsável por enriquecê-lo com informações adicionais.

## Benefícios

### Melhor qualidade das respostas

Respostas tornam-se mais relevantes.

### Maior previsibilidade

Reduz respostas inesperadas.

### Melhor experiência do usuário

As respostas ficam mais alinhadas ao contexto.

### Redução de ambiguidades

Diminui interpretações incorretas.

## Limitações

### Dependência do contexto

Pouco contexto pode gerar respostas imprecisas.

### Não substitui conhecimento externo

Prompt Engineering não resolve problemas de falta de informação.

Para isso utiliza-se RAG.

### Não altera o modelo

O comportamento do LLM continua limitado às suas capacidades.

## Resumo

Prompt Engineering é a prática de estruturar instruções para modelos de Inteligência Artificial.

Ao fornecer contexto, objetivos e restrições adequadas, é possível obter respostas mais precisas, previsíveis e alinhadas às necessidades da aplicação.
