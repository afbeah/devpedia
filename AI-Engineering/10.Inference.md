# Inference

Inference é o processo pelo qual um modelo de Inteligência Artificial utiliza o conhecimento adquirido durante o treinamento para gerar uma resposta a partir de uma nova entrada.

Seu principal objetivo é produzir previsões, respostas ou decisões utilizando um modelo previamente treinado.

Em outras palavras:

> Inference é o momento em que o modelo coloca em prática tudo o que aprendeu durante o treinamento.

## O problema

Imagine que um usuário envie a seguinte pergunta:

```txt
Explique o que é Docker.
```

O modelo já foi treinado anteriormente.

Mas como ele transforma essa pergunta em uma resposta?

É durante a Inference que isso acontece.

## Como funciona?

De forma simplificada, o processo ocorre da seguinte maneira:

```txt
Usuário
↓
Prompt
↓
Tokenização
↓
LLM
↓
Predição do próximo token
↓
Resposta
```

O modelo recebe a entrada, interpreta seu conteúdo e começa a prever, token por token, qual deve ser a próxima parte da resposta.

## Etapas da Inference

Durante uma inferência, normalmente acontecem as seguintes etapas.

### 1. Recebimento do Prompt

O usuário envia uma solicitação.

Exemplo:

```txt
Explique o que é Kubernetes.
```

---

### 2. Tokenização

O texto é dividido em pequenos fragmentos chamados **tokens**.

```txt
Explique

↓

o

↓

que

↓

é

↓

Kubernetes
```

Esses tokens são transformados em representações numéricas compreendidas pelo modelo.

---

### 3. Processamento pelo Modelo

O LLM analisa todos os tokens presentes na Context Window.

Nesse momento ele considera:

- Prompt;
- Histórico da conversa;
- Contexto do RAG;
- Instruções do sistema;
- Resultados de ferramentas.

Tudo isso influencia a resposta.

---

### 4. Predição

O modelo calcula qual token possui maior probabilidade de aparecer em seguida.

Exemplo simplificado:

```txt
Docker é uma plataforma para...
```

O modelo calcula probabilidades como:

```txt
containers
94%

máquinas virtuais
3%

servidores
2%

outros
1%
```

O token mais adequado é selecionado.

---

### 5. Geração da Resposta

Esse processo se repete centenas de vezes.

```txt
Token

↓

Novo Token

↓

Novo Token

↓

...

↓

Resposta Final
```

A resposta completa é construída um token de cada vez.

## Exemplo

Pergunta:

```txt
O que é um banco vetorial?
```

Fluxo:

```txt
Pergunta
↓

Tokens
↓

LLM

↓

Predição

↓

Resposta
```

Todo esse processo normalmente acontece em poucos segundos.

## Inference x Treinamento

Esses conceitos possuem objetivos completamente diferentes.

| Treinamento | Inference |
|-------------|-----------|
| Ensina o modelo | Utiliza o conhecimento aprendido |
| Atualiza os parâmetros | Não altera o modelo |
| Alto custo computacional | Muito mais rápido |
| Executado poucas vezes | Executado a cada pergunta |

O treinamento ocorre antes da utilização do modelo.

A Inference acontece durante cada interação.

## O que influencia a qualidade da Inference?

Diversos fatores influenciam o resultado.

### Prompt Engineering

Prompts bem estruturados produzem respostas melhores.

### Context Window

Mais contexto permite decisões mais precisas.

### RAG

Informações recuperadas enriquecem o conhecimento disponível.

### Temperatura

Controla o nível de criatividade do modelo.

Temperaturas baixas tendem a produzir respostas mais previsíveis.

Temperaturas altas tendem a gerar respostas mais criativas.

## Benefícios

### Velocidade

Modelos conseguem responder rapidamente.

### Escalabilidade

Uma mesma IA pode atender milhares de usuários.

### Flexibilidade

Pode responder diferentes tipos de solicitações.

## Limitações

### Dependência do treinamento

O modelo só utiliza o conhecimento aprendido.

### Alucinações

Pode produzir respostas incorretas.

### Custo

Inferências em modelos muito grandes exigem alto poder computacional.

## Exemplos de uso

Inference ocorre em praticamente todas as aplicações modernas de IA.

- ChatGPT;
- GitHub Copilot;
- Gemini;
- Claude;
- Assistentes Virtuais;
- Chatbots Corporativos.

Sempre que um usuário envia uma solicitação, uma inferência é realizada.

## Resumo

Inference é o processo de execução de um modelo de Inteligência Artificial.

Durante essa etapa, o LLM recebe um prompt, processa o contexto disponível, prevê os próximos tokens e gera uma resposta utilizando o conhecimento adquirido durante seu treinamento.
