# A2A (Agent-to-Agent)

A2A (Agent-to-Agent) é um padrão de comunicação que permite que diferentes agentes de Inteligência Artificial colaborem entre si para resolver problemas de forma distribuída.

Seu principal objetivo é possibilitar que agentes especializados compartilhem tarefas, informações e resultados durante a execução de um fluxo de trabalho.

Em outras palavras:

> A2A permite que múltiplos AI Agents trabalhem em conjunto para atingir um objetivo comum.

## O que significa A2A?

A sigla significa:

```txt
Agent-to-Agent
```

Em português:

```txt
Agente para Agente
```

O conceito representa a comunicação entre agentes inteligentes.

## O problema

Imagine um usuário solicitando:

```txt
Planeje minha viagem para São Paulo.
```

Essa tarefa envolve diversas atividades.

Por exemplo:

- pesquisar voos;
- reservar hotel;
- verificar previsão do tempo;
- calcular custos;
- montar um roteiro.

Um único agente poderia executar tudo isso.

Entretanto, uma abordagem mais eficiente consiste em utilizar agentes especializados.

## Como funciona?

Em vez de um único agente realizar todas as tarefas, diferentes agentes colaboram entre si.

Fluxo:

```txt
Usuário
↓
Agente Principal
├── Agente de Voos
├── Agente de Hotéis
├── Agente de Clima
└── Agente Financeiro
↓
Resposta Consolidada
```

Cada agente resolve apenas a tarefa para a qual foi especializado.

## Exemplo

Pedido:

```txt
Analise meu currículo e encontre vagas compatíveis.
```

Fluxo:

```txt
Usuário
↓
Agente Principal
↓
Agente de Currículo
↓
Competências Identificadas
↓
Agente de Vagas
↓
Vagas Compatíveis
↓
Agente de Personalização
↓
Currículo Adaptado
↓
Resposta
```

Os agentes trabalham de forma colaborativa.

## Componentes

Uma arquitetura A2A normalmente possui:

### Agente Principal

Coordena todo o fluxo.

### Agentes Especializados

Cada agente possui uma responsabilidade específica.

### Comunicação

Os agentes trocam informações durante a execução.

### Resultado Consolidado

O agente principal reúne todos os resultados antes de responder ao usuário.

## Benefícios

### Especialização

Cada agente executa apenas uma função.

### Escalabilidade

Novos agentes podem ser adicionados conforme necessário.

### Reutilização

Um mesmo agente pode ser utilizado por diferentes aplicações.

### Organização

Fluxos complexos tornam-se mais fáceis de manter.

## Limitações

### Coordenação

Quanto maior o número de agentes, maior a complexidade da comunicação.

### Latência

Cada interação entre agentes aumenta o tempo de resposta.

### Consistência

É necessário garantir que todos utilizem o mesmo contexto.

### Observabilidade

Monitorar diversos agentes simultaneamente exige ferramentas adequadas.

## Exemplos de uso

Arquiteturas A2A podem ser utilizadas em:

- assistentes corporativos;
- atendimento ao cliente;
- automação empresarial;
- pesquisa inteligente;
- desenvolvimento de software;
- análise financeira;
- sistemas jurídicos.

## A2A x AI Agent

Embora relacionados, possuem objetivos diferentes.

| AI Agent | A2A |
|-----------|-----|
| Resolve uma tarefa | Coordena vários agentes |
| Atua individualmente | Atua de forma colaborativa |
| Pode utilizar ferramentas | Pode utilizar outros agentes |
| Possui uma responsabilidade principal | Distribui responsabilidades |

Um sistema A2A normalmente é composto por diversos AI Agents.

## A2A x MCP

Esses conceitos também não são equivalentes.

| MCP | A2A |
|------|-----|
| Comunicação entre IA e ferramentas | Comunicação entre agentes |
| Integra APIs e sistemas | Integra agentes especializados |
| Conecta serviços externos | Coordena colaboração entre agentes |

MCP conecta agentes ao mundo externo.

A2A conecta agentes entre si.

## Resumo
