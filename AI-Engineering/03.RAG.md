# RAG (Retrieval-Augmented Generation)

RAG (Retrieval-Augmented Generation) é uma arquitetura utilizada para fornecer informações externas a um Large Language Model (LLM) antes da geração da resposta.

Seu principal objetivo é permitir que o modelo utilize dados atualizados, privados ou específicos de um domínio sem a necessidade de realizar um novo treinamento.

Em outras palavras:

> RAG combina a capacidade de geração de um LLM com a recuperação de informações relevantes.

## O que significa RAG?

A sigla significa:

```txt
Retrieval-Augmented Generation
```

Em português:

```txt
Geração Aumentada por Recuperação
```

O nome descreve exatamente o processo:

* Recuperar informações relevantes;
* Utilizar essas informações como contexto;
* Gerar uma resposta baseada nesse contexto.

## O problema

Imagine um chatbot treinado há alguns meses.

Pergunta:

```txt
Qual é a política de férias da empresa?
```

O LLM não possui acesso ao manual interno da empresa.

Mesmo sendo extremamente inteligente, ele não conhece documentos privados nem informações atualizadas.

Como consequência:

```txt
Usuário
↓
LLM
↓
Resposta baseada apenas no treinamento
```

A resposta pode estar incompleta ou incorreta.

## Como funciona?

O RAG adiciona uma etapa antes do LLM responder.

Fluxo:

```txt
Usuário
↓
Pergunta
↓
Busca por documentos relevantes
↓
Contexto encontrado
↓
LLM
↓
Resposta
```

Agora o modelo responde utilizando informações recuperadas da base de conhecimento.

## Exemplo

Imagine que a empresa possui um documento chamado:

```txt
Manual do Colaborador.pdf
```

Pergunta:

```txt
Quantos dias de férias um colaborador possui?
```

Fluxo:

```txt
Pergunta
↓
Busca no Manual
↓
Trecho encontrado
↓
LLM
↓
Resposta baseada no documento
```

O modelo responde utilizando o conteúdo recuperado, e não apenas o conhecimento adquirido durante o treinamento.

## Componentes de um sistema RAG

Um sistema RAG normalmente possui:

* Usuário;
* LLM;
* Base de conhecimento;
* Embeddings;
* Banco Vetorial;
* Mecanismo de busca.

Fluxo:

```txt
Usuário
↓
Pergunta
↓
Embeddings
↓
Vector Database
↓
Documentos relevantes
↓
LLM
↓
Resposta
```

## Quando utilizar?

RAG costuma ser utilizado quando o sistema precisa responder perguntas sobre:

* documentação técnica;
* políticas internas;
* contratos;
* legislação;
* artigos;
* bases de conhecimento;
* documentos privados.

## Benefícios

### Dados atualizados

Permite utilizar documentos recentes sem treinar novamente o modelo.

### Informações privadas

O modelo pode responder utilizando documentos internos da empresa.

### Menor custo

Evita processos de Fine-Tuning para cada atualização de informação.

### Melhor precisão

As respostas passam a considerar informações específicas do domínio.

## Limitações

### Dependência da busca

Se a recuperação encontrar documentos incorretos, a resposta também poderá ser incorreta.

### Qualidade da base

Documentação desatualizada gera respostas desatualizadas.

### Latência

Existe uma etapa adicional de recuperação antes da geração da resposta.

## RAG x Fine-Tuning

Embora ambos melhorem o desempenho de um LLM, possuem objetivos diferentes.

| RAG                              | Fine-Tuning                      |
| -------------------------------- | -------------------------------- |
| Recupera informações externas    | Re-treina o modelo               |
| Utiliza documentos atualizados   | Altera o comportamento do modelo |
| Mais simples de atualizar        | Processo mais custoso            |
| Ideal para bases de conhecimento | Ideal para adaptar comportamento |

## Exemplos de uso

RAG é amplamente utilizado em:

* assistentes corporativos;
* chatbots internos;
* atendimento ao cliente;
* busca em documentação;
* sistemas jurídicos;
* sistemas médicos;
* copilotos para desenvolvedores.

## Resumo

RAG (Retrieval-Augmented Generation) é uma arquitetura que combina um Large Language Model com uma base de conhecimento externa.

Antes de gerar uma resposta, o sistema recupera documentos relevantes e os fornece como contexto para o modelo, permitindo respostas mais precisas, atualizadas e alinhadas ao domínio da aplicação.
