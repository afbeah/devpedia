# Depuração

*Depuração* é o processo de identificar, analisar e corrigir erros (bugs) em um programa.

Quando algo não funciona como esperado (*ex: um botão que não faz nada, ou um cálculo errado*), usamos ferramentas de depuração para investigar o que está acontecendo dentro do código enquanto ele roda.

## Como funciona na prática?

Ao invés de adivinhar o erro, você pode:

1. Pausar a execução do programa em um ponto específico;
2. Ver o valor de variáveis naquele momento;
3. Passar linha por linha (step-by-step) para ver como o código se comporta;
4. Descobrir em que ponto o programa se comporta de forma errada.

## Ferramentas comuns de depuração

-> Em navegadores (para html/js)
  - Google chrome / Firefox / Edge
    - Pressione **F12** ou clique com o botão direito em **inspecionar**
    - Vá até a aba *"sources"* ou *"debugger"*
    - Lá você pode
      - Adicionar breakpoints (pontos onde o código para)
      - Ver variáveis em tempo real
      - Navegar o código linha a linha

-> IDEs como Vs Code / IntelliJ / Eclipse
  - Tem depuradores integrados
  - É possível:
    - marcar linhas para parar a execução;
    - acompanhar o valor de variáveis;
    - analisar chamadas de função (stack trace)


## Por quê é importante depurar?

-> Evita perda de tempo com tentativas e erros;

-> Ajuda a entender melhor como o código funciona;

-> Reduz frustrações ao lidar com bugs difíceis; 

-> Torna seu desenvolvimento mais profissional e eficiente.
